Two Moons Operation

1. Computational Aspect

The "Two Moons" dataset is a classic non-linear classification problem. On the computational side, I implemented multi-layer perceptrons (MLPs) and Kernel SVMs to effectively separate the intertwined data points. This involved optimizing the cost function using gradient descent algorithms and tuning the learning rate for convergence.

2. Analytical Aspect

Analytically, the challenge was to understand why linear models fail on this dataset. By mapping the data to a higher-dimensional space using the kernel trick, or by using the non-linear activation functions in a neural network, the decision boundary becomes linear in the transformed space. I analyzed the impact of different hyperparameters on the decision boundary's shape.

In-Depth
Predictive Modeling Course Project

Project Overview

This project focused on building a comprehensive machine learning model to predict customer churn for a telecommunications company. By identifying at-risk customers early, the company can proactively offer incentives to retain them.

The goal was to achieve high recall, ensuring we identify as many churning customers as possible, even if it means a slightly lower precision.

Methodology

1. Exploratory Data Analysis: Analyzed feature distributions and correlations.

2. Feature Engineering: Created new features based on usage patterns and tenure.

3. Model Selection: Tested Logistic Regression, Random Forest, and XGBoost.

4. Hyperparameter Tuning: Utilized GridSearchCV to optimize model parameters.

Results & Conclusion

The XGBoost model performed the best, achieving an F1-score of 0.85 on the test set. The most important features turned out to be contract type, monthly charges, and customer tenure.

By implementing this model, the hypothetical business could reduce churn rates by an estimated 15%.
