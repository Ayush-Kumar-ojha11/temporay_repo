System Requirements

Functional Requirements

FR-01

Categorization

Users shall be able to categorize the type of expense and how many people are involved in that category.

FR-02

Group Creation & Management

Users shall be able to create named expense groups, add or remove members, and manage group-level expenses.

FR-03

Add Expense

Users shall log shared expense events with details including amount, description, date, and the group members involved.

FR-04

Automatic Balance Calculation

The system shall automatically calculate individual balances — computing exactly who owes whom based on recorded expenses.

FR-05

Expense Summary View

Users shall view a consolidated dashboard showing total group expenses and their individual current balance at a glance.

FR-06

Settlement / Mark as Paid

Users shall be able to mark specific debts as settled, updating balances.

FR-07

Transaction History

The system shall maintain a complete history of all expense entries and settlement events for each group.

Non-Functional Requirements

NFR-01

Performance

The system should load pages and calculate balances within 2 sec, handling multiple simultaneous users without slowdown.

NFR-02

Security

All user data and passwords must be protected using encryption. Secure transfer sessions must be enforced.

NFR-03

Usability

The interface must be simple and intuitive for all users regardless of technical background — mobile and desktop friendly.

NFR-04

Reliability

The system should operate 99.99 percent of the time without crashes, unexpected errors, or data loss during normal usage conditions.

NFR-05

Availability

The platform should maintain 24/7 uptime, ensuring users can access and manage their expenses at any time of day.

NFR-06

Data Accuracy

All expense entries and balance calculations must be stored and computed with complete correctness — no rounding errors.

NFR-07

Maintainability

Codebase should be modular, well-documented and structured to allow easy updates, feature additions and bug fixes.

NFR-08

Compatibility

The web application must function correctly across major browsers (Chrome, Firefox, Safari) on mobile, tablet, and desktop.

NFR-09

Scalability

The architecture must support growth in users and groups without requiring significant redesign of core system components.
